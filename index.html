Multi-Agent Text-to-SQL & Analytics Engine

GitHub Live Demo

Context

Enables non-technical teams to query secure enterprise relational databases in natural language. Translates English to executable SQL. Key engineering challenge: preventing LLM hallucinations from generating destructive queries or querying unauthorized schemas, and routing complex queries to specialized sub-agents.

Security & Tradeoffs

Implements Abstract Syntax Tree (AST) validation to block non-SELECT statements and schema leaks before execution. Groq LLaMA-3 serves as the primary router due to low latency (sub-500ms), while query generation falls back to Claude-3 for multi-table joins. Query execution runs in an ephemeral, read-only Docker container to isolate potential execution vulnerabilities.

Evaluation & Metrics

94.8% text-to-SQL accuracy <1.8s end-to-end latency Zero sandbox escapes

Infrastructure & Deployment

Docker containers · FastAPI endpoints · SQLite sandbox (demo) · LangChain multi-agent execution graphs · LangSmith telemetry.

Clinical Screening CNN Pipeline

GitHub Model Card

Context

A computer vision pipeline engineered to scan chest X-rays and automatically flag critical pneumonia cases for priority triage. Designed to reduce patient queue times in resource-constrained clinics. In clinical screening, a false negative is lethal, making high sensitivity (recall) the primary optimization objective.

Optimizations & Tradeoffs

Addressed heavy class imbalance (normal vs. pneumonia) using focal weighted-loss vectors in training rather than synthetic oversampling. Compiled and quantized the ResNet-50 backbone to INT8 using ONNX Runtime. This traded a negligible 0.15% classification score for a massive 4.2x speedup, making edge CPU inference feasible.

Evaluation & Metrics

96.1% clinical recall (sensitivity) 94.2% accuracy 12ms ONNX CPU latency

Infrastructure & Deployment

Triton Inference Server · Docker · ONNX Runtime CPU execution · Grad-CAM feature localization layers · Prometheus drift monitoring.

Energy Grid Demand Forecasting Engine

GitHub API Spec

Context

Predicts electricity load on regional grids at 30-minute intervals up to 72 hours in advance, allowing grid operators to optimize power plant output and prevent blackouts. Moving from simple point predictions to robust risk bounds was essential for operational deployment.

System Design & Conformal Prediction

Ensembles statistical baselines (Prophet/ARIMA) for long-term seasonality with deep learning (LSTM) and boosting (XGBoost) models to capture non-linear, weather-driven residual spikes. Rather than raw point forecasts, the system implements Conformal Prediction methods, guaranteeing a mathematically sound 95% confidence interval boundaries for load planning.

Evaluation & Metrics

2.8% MAPE (Mean Absolute % Error) 95% Conformal Coverage rate 72h forecast horizon

Infrastructure & Deployment

FastAPI service · TimescaleDB time-series database · Prefect workflow orchestration · MLflow model registry · PyTorch & XGBoost.

Enterprise Hybrid-Search RAG Pipeline

GitHub Architecture Spec

Context

A question-answering pipeline built for dense engineering codebases and API documentation. Traditional semantic vector search fails on exact lexical hits like code symbols or logs. This engine couples semantic vectors with sparse token search to achieve production-grade retrieval precision.

Optimizations & Latency Caching

Implements hybrid search (dense BGE-M3 + sparse BM25) fused using Reciprocal Rank Fusion (RRF) and re-ranked using Cohere Rerank v3. To prevent expensive, repetitive LLM calls, a local Redis-based semantic cache (GPTCache) computes cosine similarity of incoming query embeddings, serving cache matches in sub-80ms while cutting API usage costs by 32%.

Evaluation & Metrics

91.2% retrieval precision (RRF) 32% API token cost savings <80ms semantic cache hit latency

Infrastructure & Deployment

Pinecone vector database · Redis (GPTCache) · Cohere Rerank API · FastAPI · LangChain validation framework.

Real-time Credit Delinquency Risk Engine

GitHub Deployment Dashboard

Context

Processes high-throughput transaction telemetry and account metadata to predict customer loan delinquency risks 30 days before a default event. Provides collections teams with early actionable risk warnings. Risk teams require local explanations for auditing, meaning raw model scores alone are insufficient.

Feature Engineering & Explainability

Integrates a central Feast Feature Store to serve transactional aggregations with low latency (<5ms). Solves extreme class imbalance by embedding SMOTE validation within localized cross-validation folds (avoiding data leakage). The model (XGBoost) calculates individual SHAP values in parallel during scoring, returning feature attribution JSONs alongside credit risk probabilities.

Evaluation & Metrics

0.894 AUC-ROC <25ms API response latency 30-day foresight horizon

Infrastructure & Deployment

FastAPI endpoints · Feast Feature Store · Redis online store · MLflow tracker · Prometheus monitoring Population Stability Index (PSI) and input distribution drift.

[2025.03]

Semantic chunk overlap vs fixed-size chunking for RAG

Broke context across technical definitions when using naive character splitting. Model hallucinated on technical specifications.

Switched to paragraph-level chunking with a 15% sliding overlap. Preserved sentence integrity and document hierarchies.

[2025.02]

SMOTE before train-test split on credit risk dataset

Evaluation looked outstanding (99% recall). In reality, synthetic minority samples leaked from the training set into validation.

Restructured pipeline to split folds first. Apply SMOTE strictly within training loops. Corrected evaluation to a realistic 89.4% AUC.

[2025.01]

Accuracy optimization for clinical pneumonia detection

Model predicted pneumonia for almost all cases due to class imbalance, achieving 76% accuracy but zero diagnostic value.

Rebuilt objective function around recall-weighted cross-entropy loss. Prioritized catching false negatives in patient triage.

[2024.11]

OpenAI API for the Text-to-SQL analytics engine

API execution cost scaled linearly with schema size, resulting in ~$0.008 per query. Unfeasible for high-volume enterprise telemetry.

Moved routing to local Groq LLaMA-3. Reduced query latency to sub-3s and lowered operational API expenses by 90%.

[2024.09]

Prophet point-forecasts for energy grid load planning

Prophet missed sharp, non-linear spikes caused by sudden heatwaves, leading to costly energy under-provisioning risk.

Ensembled Prophet baseline with XGBoost and LSTM residuals. Implemented Conformal Prediction to produce mathematically safe 95% risk bands.

[2024.07]

Fine-tuning a local model for domain-specific Q&A

Training times were high, and the model still hallucinated facts when asked about out-of-distribution corporate documents.

Replaced with a retrieval-augmented generation (RAG) architecture. Fine-tuning should shape style, RAG should supply grounding.
